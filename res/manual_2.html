1. General

Main sections of the Mesecina user interface:

  • Vizualization widget - the biggest part of the main window, is a viewport to a scene in which one can navigate to study the geometric structures.
  • Layers - a list of available geometric structures which can be included in the scene. Whenever a layer is checked, the corresponding geometric structures are going to be introduced in the scene. They are computed lazilly, the first time they are requested, but after that they are stored in the memory for fast interaction.
  • Bundles - bundles are collections of layers. They are means to store scene settings which are often used. By clicking on a bundle one can activate it and set the layer configuration which was saved into the bundle
  • Slider - the big slider on the top is used for procedural manipulation of the geometric structures. One can use an evolution in the combo box and then use the slider to apply the corresponding evolution on the geometries. Again lazy computation is available, only layers which are selected are going to be computed, but at any time one can compute additional structures using the
  • Record - the menu and the button on the top of the main window allows you export images, or sequences of images of the scene which is currently viewed
  • Kernel - one can choose the arithmetic kernel used for all geometric algorithms. The double is faste but certain algorithm will fail returning a useful result. The CORE::Expr kernel's exact arithmetic is slower but computes precise results.

Note that you can resize all the sections of the mainwindow by moving the splitter edges. All your settings regarding window positions, sizes, splitters will be saved between application settings

2. Mouse and Navigation

Navigation, point insertion and manipulation are all done with the mouse and the CTRL, ALT, SHIFT button combinations. This allows for very fast control of the application as soon as one got used to the controls

2.1. Navigation (mouse + CTRL)

CTRL + left drag pan
CTRL + right drag zoom in/out
CTRL + middle drag zoom to a region of the screen smoothly
CTRL + middle click zoom to bounding box smoothly
CTRL + ALT + middle drag zoom to a region of the screen instantly
CTRL + ALT + middle click zoom to bounding box instantly

2.2. Point insertion

left click insert a single point
middle drag "spray" points along the path of the mouse
ALT + left click or left drag same as above above but the actual computation on the geometry structures is deferred until ALT is released

2.3. Selection and modification (mouse + SHIFT)

SHIFT + left click on a point select/deselect that point
SHIFT + middle drag add points in a rectangular region to selection
CTRL + SHIFT + left drag move selection with continuous update
ALT + CTRL + SHIFT + left drag move selection with update only on release of ALT
DEL or backspace delete selection

3. Layers and Bundles

Layers represent geometric structures and when they are checked then they are visualized in the scene. If the structures needed for the layer are not available in the memory they will be computed when the used checkes them in. Notice that layers which were already computed and are in the memory will be displayed with darker color in the layer's list.

One can change the properties od the layers by rightclicking on their name and using the available options. One can change color, opacity, point size, line width and such properties.

The order of the layers determine in which order are they rendered in the scene. The lower a layer is in the list the later it will be rendered, therefore the "closer" it will be the user. Use SHIFT + drag and drop on the list to reorder layers.

After configuring a scene with several layers and properties one can save the whole configuration in a bundle for later use. For this use the combo box and the button on the bottom of the layers list. Once a bundle saved it will be introduced in the bundles list and saved on the hard disk, so one can use bundles across application sessions. To activate a layer simply click on it in the bundles list. One can reorder (SHIFT + drag and drop) or rename bundles (doubleclick on the name). Right clicking on a bundle brings up a context menu, where one can delete or export a bundle to a file.

One can import bundles from files in the Bundle -> Import bundles... menu, or one can export all available bundles using Bundles -> Export bundles...

5. Evolution Sliders

The slider allows to study evolution of the structures when certain operations are applied on primitives. The combobox value specifies which evolution does the slider refer to the slider controls which step of the evolution do we see currently in the visualization widget.

The textfield on the right allows to change the resolution of the slider. The two buttons will move automatically the slider to the current maximum left or right location on the slider. It can be used to trigger batch computations or view animations after structures have been computed.

Note the initial dataset for all evolution is the same , the last state before the first evolution step computed. Whenever a dataset is changed (move, delete, insert) the slider will be reset, all the evolutions are lost.
